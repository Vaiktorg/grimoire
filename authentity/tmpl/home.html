Simple Web Identity

A KISS approach to zero-trust identity model and authoring service_

Identity

Account and Identity Management: Authentity provides services for account creation, identity registration, and account retrieval.

Authentication

Authentity uses GWTs; A compact, URL-safe JWT variation. An easier way to represent your authentication structure shared between two parties.

Authorization

Authorization and Resource Access is achieved via middlewares which ensures access with an extensible RBAC (Role-Based Access Control).

Authenticity

Validate access by the correct Issuer and proper receive of the Recipient via customizable Visual Web Tokens (VWT)

Safe / Secure / Private

Share What You Want: You decide how much info you want to provide your identity. It can be a whole portfolio piece, or a contact card. Authentity is your passport to the internet.

Role Based Access Control: Control access based on general application of Resource, Role, Claims and Permissions. You can even control access based on required Identity information.

Secure Password Handling: Secure password handling during the login process.

Text & Visual Authorization: Authenticate with the use of header token or visual passport which can be used temporarily.

User Management: Allows for user management through features like account and profile handling.

Error Handling: Includes error handling and recovery.

Analytics: CPU, Threads, Memory, and more can be used to maintain app performance and stability. Logging included for debugging and audit purposes.

Context-Aware Operations: Contextualized operations for handling and propagating values, cancelation signals, and deadlines across APIs and processes from request level.
